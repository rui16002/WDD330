Findings

Function properties and methods

Functions have properties like "lenght" that indicates the number of parameters

Call and apply

The call() method can be used to set the "this" variable inside a function.

The apply function works the same way, but parameters are added as arrays.

Custom properties

It is interesting how we can add custom properties to functions such as function description

Immediately invoked functions

I am a little familiar with this, I have seen it applied mostly when dealing with callbacks. We have to be carefull though not to use it excesively as code can easily turn into spaguetti code.

It could be used with temporary variables and initialization functions. Althought in ES6 this could be replaced by code blocks.

It could also be used to safely use strict mode (on our own code). To write functions that rewrite themselves(This is a powerful feature). We have to be careful with defined properties as they need to be redefined as well, otherwise, they will result in undefined.

Init-time branching, this is also very powerful as we can define a function given a certain environment and use it without the need to check the environment everytime

It could also be used for recursive functions

Error first callback pattern

this is the pattern used in nodeJS

Promise

promises solve the same problems callbacks do in a more readable ordered way. Promises are created with a constructor that takes an executor function with two function parameters, resolve and reject. Once a promise has been settled, we can deal with the outcome using then(fulfilmentFunction, rejectionFunction) or then(fulfilmentFunction).catch(rejectionFunction). Promises can be chained together to improve readability.

This behavior can also be achieved with async and await, these two reserved words help to write asynchronous code as if it was syncronous.

Functions can return functions

Generators

Generators are functions that creates iterators, they use an asterix after function and yield instead of return. They use the next() function to get the next value produced by the yield command

Multiparadigm programming

JS can be used with various programming paradigms, functional programming, procedural programming and OOP. Even a combination of them.

Pure functions

functions that return a calculation based on the parameters passed without altering them.

High order functions

High order functions are functions that receive or return other functions

Currying

Partial application of a function given the parameters passed

AJAX

Ajax is a technic to send and receive data to/from the server asynchronously, ussually as a JSON file. This is usefull to do partial page updates. Ajax requests can be made using the Fetch API. There are several interfaces that allows us to control requests and responses (Headers, Response, Request).

The FormData interface makes it easier to send data from forms.
