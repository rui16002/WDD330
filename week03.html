I used javascript before but I thought that it was just used for linear programming, I did not know it was possible to create and manage objects.

I have to be careful with creating an array with the array constructor using just one element, in javascript, this is interpreted as the size, so, we would have unexpected behaviours.

I like ES6 notation for classes, it is much more familiar for those writing code in another language.

Objects can morph at runtime using the prototype property.

Monkey patching ("Extending" built in classes with new functionality) seems to be a very powerfull tool, it is a shame that it is not recommended in JS

When creating objects from objects, intance of won't work, isPrototypeOf should be used instead

Borrowing methods from prototypes is an interesting concept. JS is very flexible

Modules seems to be more effective than libraries, when the latter is just a collection of functions
