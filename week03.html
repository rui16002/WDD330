Patterns and regular expressions are great to validate inputs before sending to services

Inputs should be validated on both ends as they can be manipulated on client side

JSON format is like XML but simpler

DOM (Document object Model) represents the HTML. It could be accessed and manipulated through JS

Event listeners help us to react to different user events (actions) and manipulate the page content/behaviour accordingly

Event handlers are defined inline in the HTML, EG: on a button onclick property: onclick="function"

Event listeners are defined in js files or tags, EG: document.body.addEventListener("click", doSomething);

There are several types of events, EG: On click, onblur, onload, mouseup, mousedown, mouseover, keyup, keydown

preventdefault function stops default behaviour from happening EG: information being sent to the service on submit a form
