Findings

Transformations

The translate(x,y) function moves an element x from the left, and y from the top. EG: transform: translate(45px, -45px);

The scale(x,y) function scales an element by the defined factors horizontally then vertically. EG: transform: scale(1.5, 0.25);

The rotate() function rotates an element around the point of origin by a specified angle value

The skew(x,y) function specifies a skew along the x and y axes. EG: transform: skew(15deg, 4deg)

The transform-origin property moves the rotation point of the object to the one specified.

The order of transforms DOES matter

Transitions

The transition-property property defines the CSS properties of the element that should be transitioned, with all for all properties being the default.

The transition-duration property sets how long the transition will take. EG: transition-duration: 0.2s;

The transition-timing-function lets you control the pace of the transition in even more granular detail.The most common timing functions include the key terms ease , linear , ease-in , ease-out , or ease-in-out

The transition-delay property allows to introduce a delay before the transition begins. EG: transition-delay: 50ms;

The transition property let us group all four in just one line like so: transition: transform 0.2s ease-out 50ms;

we can also group transitions like so: transition: transform 0.2s ease-out 50ms, color 0.1s linear 50ms;

Keyframes - animations

To animate an element in CSS, you first create a named animation, then attach it to an element in that element’s property declaration block. EG: @keyframes appearDisappear { 0%, 100% {opacity: 0;} 20%, 80% {opacity: 1;}}

As with transitions, some properties need to be added: animation-name, animation-duration, animation-timing-function

he animation-iteration-count property lets you define how many times the animation will play through: EG: animation-iteration-count: infinite;

On multiple iterations animations I would want to pay attention to the animation-direction property. Instead of jumping abruptly to the beginning I can start the second iteration from 100% to 0%

The animation-play-state property defines whether the animation is running or paused

The animation property is a shorthand for all the properties mentioned above like so: animation: 300ms ease-in infinite alternate 5s backwards appearDisappear running;

Canvas

We can draw a blue rectangle with red stroke like so: context.strokeStyle = "red"; context.fillStyle = "rgba(0, 0, 255, 0.5)"; context.fillRect(10, 10, 100, 100); context.strokeRect(10, 10, 100, 100);

Notice that the canvas coordinate system goes top-to-bottom and left-to-right

We can use the drag and drop API to allow certain elements to be dragged and dropped. We need to define an attribute (draggable="true") to the elements (The rest of the elements don't need to have this property set to false, since draggable is not boolean)

we need to define an event listener to deal with the dragging: addEventListener('dragstart', function (event) { event.dataTransfer.setData("text/plain", this.id); });
